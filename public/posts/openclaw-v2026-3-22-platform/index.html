OpenClaw v2026.3.22 深度解析：AI Agent 从「工具」向「平台」的质变

300+ 项改动、30+ 名开发者贡献，这次更新为什么被称为「需要单独目录的版本」？

一个看似普通的版本号#

3 月 22 日，开源 AI Agent 项目 OpenClaw 发布 v2026.3.22 版本。

表面上看，这只是一次常规更新——但在开发者社区，它被称为：

「需要单独目录的更新版本。」

原因很简单：300+ 项改动、30+ 名开发者参与贡献，更重要的是，带来了几个结构性变化。

很多人认为，这个版本标志着 OpenClaw 从一个工具，开始走向一个平台。

最大变化：插件市场 ClawHub 正式上线#

本次更新最核心的功能，是一个叫 ClawHub 的东西。

简单来说：ClawHub = OpenClaw 的插件市场。

此前，用户想给 Agent 增加能力，通常需要：

  • 自己去 GitHub 搜索插件
  • 手动安装 npm 包
  • 再逐一配置到 Agent 系统

整个流程门槛较高，对普通用户并不友好。

而在 v2026.3.22 中，只需两行命令：

openclaw skills search
openclaw skills install

社区贡献的技能模块，可以像安装 App 一样直接获取和使用。

这个变化的意义不只是「更方便」，而是改变了能力供给的方式——Agent 的能力边界，不再只由官方团队决定，而是由整个开发者生态共同扩展。

平台 系统 生态
iPhone iOS App Store
Android Android Play Store
OpenClaw Agent 系统 ClawHub

这也是为什么很多开发者认为：OpenClaw 正在进入「Agent App Store」阶段。

每个 Agent 可以拥有不同的「大脑」#

新版本引入了 Per-agent Reasoning 机制：在同一套系统内，每个 Agent 可以独立绑定不同的底层模型，例如 MiniMax M2.7、GPT-5.4-mini、GPT-5.4-nano 等。

这意味着一套 Agent 系统内部，可以形成「分工结构」：

Agent 绑定模型 负责任务
Research Agent MiniMax M2.7 搜索与研究
Planner Agent GPT-5.4-mini 制定计划
Worker Agent GPT-5.4-nano 执行任务
Monitor Agent GPT-5.4-nano 监控系统

这种模式有点像企业组织架构：复杂任务用强模型，简单任务用轻量模型，在成本控制和任务效率之间找到平衡。

Agent 开始具备「远程操作能力」#

新增的 OpenShell + SSH Sandboxes 组件，允许 Agent 在受控沙箱环境中通过 SSH 执行远程操作，包括：

  • 登录服务器
  • 执行系统命令
  • 部署代码
  • 管理服务

过去，OpenClaw 更多作为本地自动化工具使用。这次更新之后，它开始向 DevOps 自动化、服务器管理等场景延伸。

所有操作均在沙箱内隔离执行，具备基础的安全边界。

搜索能力升级：向「研究型 Agent」靠拢#

v2026.3.22 集成了多个搜索与抓取工具：

  • Exa —— AI 语义搜索
  • Tavily —— 实时信息查询
  • Firecrawl —— 网站数据抓取

组合起来后，Agent 可以自动完成：

搜索资料 → 抓取网页 → 汇总信息 → 生成报告

这类端到端的研究能力，被很多开发者称为「Research Agent」的雏形。

安全策略同步升级#

插件市场的上线，也带来了供应链安全的讨论。

社区此前曾出现过数百个恶意插件，可能窃取 API Key、读取本地文件，甚至执行远程代码。

为此，v2026.3.22 将默认权限收窄：

tools.profile = messaging

也就是说：Agent 默认不具备系统级权限。 文件操作、系统命令等高权限行为，需要用户手动开启。

官方同时建议：

  • 优先使用经过审核的插件
  • 在沙箱环境中运行自动化任务
  • 定期检查插件来源

一个更大的趋势#

把这些变化放在一起看，会看到一条更清晰的路径：

OpenClaw 正在构建的，是一套完整的平台结构：

Agent 系统 + 插件生态 + 多模型架构 + 远程执行能力

这种结构并不陌生。操作系统需要应用商店，平台需要开发者生态——当生态形成之后，能力的供给方式就会发生根本性变化。

AI Agent 是否会走出同样的路径，目前还是开放问题。但 OpenClaw 这次更新，至少给出了一个具体的方向：

AI Agent 正在从实验工具，逐渐走向基础设施。

小结#

OpenClaw v2026.3.22 的核心变化，用五个关键词概括：

  • ClawHub：插件市场，Agent 能力可安装、可扩展
  • Per-agent Reasoning：每个 Agent 独立绑定模型，支持分工架构
  • SSH Sandboxes：沙箱内远程执行，面向 DevOps 场景
  • 搜索集成：Exa / Tavily / Firecrawl，Research Agent 雏形
  • 安全升级：默认 messaging 权限，供应链风险收口

工具和平台之间，隔着一个生态。OpenClaw v2026.3.22 还在早期，但它正在铺设的基础设施，值得持续关注。
