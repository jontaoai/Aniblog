OpenClaw v2026.3.22 深度解析：AI Agent 从「工具」向「平台」的质变

300+ 项改动、30+ 名开发者贡献，这次更新为什么被称为「需要单独目录的版本」？ 一个看似普通的版本号 3 月 22 日，开源 AI Agent 项目 OpenClaw 发布 v2026.3.22 版本。 表面上看，这只是一次常规更新——但在开发者社区，它被称为： 「需要单独目录的更新版本。」 原因很简单：300+ 项改动、30+ 名开发者参与贡献，更重要的是，带来了几个结构性变化。 很多人认为，这个版本标志着 OpenClaw 从一个工具，开始走向一个平台。 最大变化：插件市场 ClawHub 正式上线 本次更新最核心的功能，是一个叫 ClawHub 的东西。 简单来说：ClawHub = OpenClaw 的插件市场。 此前，用户想给 Agent 增加能力，通常需要： 自己去 GitHub 搜索插件 手动安装 npm 包 再逐一配置到 Agent 系统 整个流程门槛较高，对普通用户并不友好。 而在 v2026.3.22 中，只需两行命令： openclaw skills search openclaw skills install 社区贡献的技能模块，可以像安装 App 一样直接获取和使用。 这个变化的意义不只是「更方便」，而是改变了能力供给的方式——Agent 的能力边界，不再只由官方团队决定，而是由整个开发者生态共同扩展。 平台 系统 生态 iPhone iOS App Store Android Android Play Store OpenClaw Agent 系统 ClawHub 这也是为什么很多开发者认为：OpenClaw 正在进入「Agent App Store」阶段。 ...
