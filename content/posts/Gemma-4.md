## 它到底强在哪里？

Gemma 4 继承了 Gemini 系列的研究积累，重点优化了一个指标：**智能密度（intelligence-per-parameter）**——用更少的参数，跑出更强的效果。

几个关键能力：

**多模态输入**
支持文本、图像输入，部分尺寸还支持音频和视频，不只是文字模型。

**超长上下文**
工作站级模型最高支持 256K 上下文，边缘模型也达到 128K。一次处理整本书、整个代码库，不是问题。

**原生函数调用与结构化输出**
内置函数调用能力，不需要额外 Prompt 工程，直接对接 Agent 和工作流。

**高效架构**
引入 Mixture-of-Experts（MoE）和 Per-Layer Embeddings 技术，大模型在有限硬件上跑得更快、更省显存。

**基准表现**
在 Arena AI 聊天竞技场等评测中，31B 和 26B 版本在推理、编码和 Agent 任务上表现出色，指令调优模型尤其亮眼。

## 四个模型怎么选？一张表看懂

Gemma 4 分两类、四个尺寸，定位非常清晰：

| 模型 | 有效参数 | 总参数 | 适合场景 | 硬件要求 |
|---|---|---|---|---|
| E2B | 2.3B | 5.1B | 手机、浏览器、低功耗设备 | 普通笔记本/手机 |
| E4B | 4.5B | 8B | 移动端多模态任务 | 普通笔记本/手机 |
| 26B MoE | ~3.8B激活 | 25.2B | Agent、高吞吐量推理 | 16-32GB 显存起步 |
| 31B Dense | 31B | 31B | 复杂推理、长上下文 | RTX 4090/5090 及以上 |

**选择逻辑：**

- 手机或低配笔记本 → **E2B / E4B**，轻量好用
- 个人工作站、有独显 → **26B MoE**，速度快、效果强，性价比最高
- 追求极限质量、不差显存 → **31B Dense**，复杂任务首选

> Google 官方明确：本地运行时，数据不离开你的设备，隐私完全由你掌控。

## 2 分钟上手：Ollama 一键部署

最快的体验方式就是 Ollama（支持 Mac、Windows、Linux）。

**前提：Ollama 版本需在 0.20 及以上。**

打开终端，直接输入：
```bash
# 边缘轻量版
ollama run gemma4:e2b
ollama run gemma4:e4b

# 工作站强力版
ollama run gemma4:26b
ollama run gemma4:31b
```

运行后，直接像聊天一样交互。支持多模态时，可以上传图片或描述场景进行测试。

**进阶玩法：**

- 通过 Ollama API 接口，把 Gemma 4 接入自己的代码、网页或 Agent 框架
- 社区已提供 GGUF 量化版本，进一步降低显存占用
- Hugging Face 和 Kaggle 也已上线模型权重，支持微调和灵活部署

## 本地 Gemma 4 能做什么？

**本地代码助手**
原生函数调用能力，写脚本、调试代码、生成结构化输出，不需要联网。

**多模态理解**
上传图片分析图表、描述场景，或处理简单音频转录，一个模型搞定。

**Agent 工作流**
结合开源 Agent 工具，搭建本地自动化流程（文档总结 + 任务编排），数据不出本地。

**长文档处理**
256K 上下文支持下，一次性处理整本书或超长文档，深度总结、全文问答都可以。

**边缘设备创新**
手机跑轻量版，实现离线智能助手、实时图像理解，真正的端侧 AI。

> 开发者社区已有大量反馈：用 Gemma 4 26B MoE 在本地跑 Agent，速度和效果都让人惊喜，尤其适合注重隐私的企业内部工具或个人项目。
